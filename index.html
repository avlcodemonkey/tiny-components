A small web component library built using Lit with Chota for styling. The goal is create a set of lightweight components that provide the core functionality needed for a typical CRUD app. Inspect this page to see how each component is used. This library is best used with a server-side rendered application (using something like Rails or DotNetCore Razor) that needs a bit more client side interactivity than the browser can natively provide.

The navigation menu above uses the lit-nav and lit-tab tags. lit-nav has a single boolean attribute is-full to control whether the nav menu is the full width of the container. The inner content of each lit-tab is displayed in the nav menu, and each lit-tab has a key attribute that specifies the unique key for that tab. The content displayed for each tab is contained in a div with a slot attribute value that corresponds to a key from the lit-tab. lit-tab can also have a boolean is-active attribute to specify that tab should be displayed by default. If no active tab is specified, the first tab is displayed by default. If multiple tabs have is-active, the first is displayed by default.

Alerts are created using the lit-alert tag. The inner content is displayed within the alert.

lit-alert Attributes
type
Controls the styling of the alert. Accepts a value of success or error. Defaults to success. An alert with an invalid type value will render nothing.
no-dismiss
Boolean that prevents the alert from being dismissed.
lang
Language for i18n. Accepts value of en or es. Defaults to en.

type
Controls what type of modal is displayed. Accepts a value of dialog or confirm. A modal with an invalid type value will render nothing.

lang

lit-table renders JSON data in table format. It provides common table functionality including paging, filtering, and multi-column sorting. lit-table uses sessionStorage to remember these settings and restores the table to it's previous state if the user navigates away and back. This component is designed to handle the most common use cases for a table in a CRUD app. But it is somewhat simplistic in that it does not offer complex rendering/formatting options currently.

lit-table Attributes
key
Unique identifier for the table. Required to use sessionStorage, otherwise optional.
src
Specifies the URL to fetch JSON table data from.
row-key
Name of the property within the JSON data that uniquely identifies each row.
add-url
Optional URL for adding records. If provided, an add button is displayed in the header of the left-most table column.
edit-url
Optional URL for editing records. If provided, an edit button is displayed in the left-most table column for each row.
delete-url
Optional URL for deleting records. If provided, a delete button is displayed in the left-most table column for each row. A modal is used to confirm before redirecting to the URL.
lang
Language for i18n. Accepts value of en or es. Defaults to en.

Both the edit and delete URL properties can include values from the JSON data that are replaced when rendering the table. For example, the URL https://localhost/edit?id=${id} would replace ${id} with the value of the id property for each row.

Each lit-table contains one to many lit-table-header tags that define the columns to display in the table. The inner content of the tag is displayed in the table header. The property attribute specifies the property in the JSON data to display for the column.
